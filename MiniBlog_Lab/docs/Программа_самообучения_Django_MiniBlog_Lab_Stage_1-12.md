# Программа самообучения Django (Stage 1--12)

## Учебный проект: MiniBlog Lab

> Автор программы: ChatGPT\
> Цель: не просто научиться писать Django-код, а понять архитектуру
> фреймворка и научиться самостоятельно строить проекты.

------------------------------------------------------------------------

# Общая идея

Вместо простого повторения уроков ты будешь постепенно строить
собственный проект **MiniBlog Lab**.

Каждый Stage включает:

-   цель;
-   теорию;
-   практику;
-   обязательные эксперименты;
-   контрольные вопросы;
-   критерии завершения;
-   Git-коммит.

Главное правило: **сначала понять, потом писать код.**

# Stage 1. Создание проекта

### Цель

Понять: - что делает `django-admin`; - зачем нужен `manage.py`; - что
такое конфигурационный пакет (`config` или `blog`); - как запускается
сервер разработки.

### Практика

-   создать виртуальное окружение;
-   установить Django;
-   выполнить `django-admin startproject config .`;
-   запустить `python manage.py runserver`;
-   выполнить `python manage.py check`;
-   изучить структуру проекта.

### Эксперименты

-   запустить сервер на другом порту;
-   посмотреть `python manage.py help`;
-   изучить `settings.py` и `urls.py`.

### Контрольные вопросы

1.  Чем `django-admin` отличается от `manage.py`?
2.  Что создаёт `startproject`?
3.  Для чего нужен `settings.py`?

### Git

``` bash
git commit -m "feat: create Django project"
```

-   выполнить `python manage.py check`.

### Контрольные вопросы

-   Что такое Django App?
-   Почему один проект может содержать несколько приложений?

# Stage 3. URL и View

    ↓
    URL
    ↓
    View
    ↓
    HttpResponse

### Практика

-   подключить его через `include()`;
-   создать страницы:
    -   /

### Эксперименты

-   открыть несуществующий URL;
-   изменить маршруты.

### Контрольные вопросы

-   Что делает `include()`?
-   Что получает View в аргументе `request`?

# Stage 4. Templates

### Цель

Научиться отделять HTML от Python.

### Практика

Создать: - base.html - home.html - about.html - contacts.html

Изучить: - extends - block - context - render() - template tags -
template variables

### Эксперименты

-   вызвать `TemplateDoesNotExist`;
-   удалить переменную из context;
-   использовать `{% if %}` и `{% for %}`.
